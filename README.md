# 190x4 PC Control

Удалённое управление Windows-ПК из Telegram-бота и Android Hub 190x4.

## Архитектура v2

- `agent-dotnet/` — self-contained Windows-агент на .NET 8.
- Персональный ключ каждого ПК хранится через Windows DPAPI в фиксированном общем
  каталоге Public Documents, поэтому запуск от администратора и обычного пользователя
  использует один конфиг.
- Автозапуск выполняется штатно через `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`,
  чтобы агент работал в интерактивной сессии рабочего стола.
- Команды доставляются по WSS и имеют UUID, срок жизни и ACK.
- Сопряжение нового ПК подтверждается владельцем через Telegram.
- Live-экран передаётся бинарными JPEG-кадрами по WSS.
- OBS управляется через локальный WebSocket; записи загружаются потоково.
- Переход со старого PowerShell-агента выполняется плавно, без остановки управления.

## Установка

Пользователь скачивает один файл `PCControl-Setup.exe`, запускает его без повышения
прав администратора и вводит свой Telegram ID. Установщик распаковывает
self-contained агент в `%LOCALAPPDATA%`, регистрирует его в автозапуске текущего
пользователя и запускает сопряжение. После этого пользователь
подтверждает ПК в Telegram.

В установленной системе NirCmd и отдельный PowerShell-скрипт не требуются.

## Сборка

Сборка выполняется только в GitHub Actions на `windows-latest`:

1. `dotnet publish` создаёт единый `PCControlAgent.exe`.
2. `build/embed.ps1` встраивает агент в PowerShell-шаблон установщика.
3. `ps2exe` создаёт один распространяемый `PCControl-Setup.exe`.

- `workflow_dispatch` — проверочная сборка и артефакт.
- тег `vX.Y.Z` — GitHub Release с установщиком и агентом.

## Миграция старых установок

Legacy-bootstrap распространяется сервером обновлений отдельно и намеренно не
публикуется в этом репозитории: старые установки могут содержать персональные
параметры OBS. При запуске версии 2.0.8 агент сам регистрирует автозапуск текущего
пользователя и удаляет старые задачи `PC Control Bot` и `190x4 PC Control`, если у
текущей учётной записи есть права на их удаление. При миграции он выбирает
конфигурацию с действующим токеном среди всех старых расположений, сохраняет
атомарный снимок конфигурации перед самозаменой бинарника, удаляет старые задачи
до загрузки конфига и периодически проверяет
доступность обновлений даже при постоянном WSS-соединении.
